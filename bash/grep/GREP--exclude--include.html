GREP--исключить /--включить синтаксис (делать не grep через определенные файлы)

Я ищу для строки "foo =" (без кавычек) в текстовых файлов в дереве каталогов. Это на общих Linux машине, у меня есть оболочки bash:

grep -ircl "foo=" *

В каталогах, также являются многие двоичные файлы, которые сопрягают "foo =".Как эти результаты не являются актуальными и замедляют поиск, я хочу grep пропустить поиск этих файлов (главным образом, JPEG и PNG изображения): как это будет делать?

Я знаю есть exclude = план и--включают = план варианты, но то, что формат шаблона?manpage grep говорит: 

--include=PATTERN     Recurse in directories only searching file matching PATTERN.
--exclude=PATTERN     Recurse in directories skip file matching PATTERN.

Поиск по включают grep, grep включают исключить, исключить grep и варианты не нашел что-либо отношение

Если есть способ лучше оглавлению только в определенных файлов, я все для него; перемещение нарушителя файлов не является альтернативой, не может найти только определенные каталоги (структура каталога является большой беспорядок, все во всем мире).Кроме того я не могу установить что-нибыдь, так что я должен делать с общие инструменты (например, grep или было предложено найти).

ОБНОВЛЕНИЯ: @ Адам Rosenfield в ответ, как раз то, что я искал:

grep -ircl --exclude=*.{png,jpg} "foo=" *

@ rmeador в ответ также является хорошим решением:

grep -Ir --exclude="*\.svn*" "pattern" *

Поиск рекурсивно, игнорирует двоичные файлы и не заглянуть внутрь Subversion скрытые папки.(...)
